修改beef的配置文件，此处只做示例，详细情况根据自己的环境和想法修改。beef配置文件采用yaml格式，内部有注释，通俗易懂。

/usr/share/beef-xss/config.yaml

在最后的扩展部分找到msf设置为true，其余的端口密码，路径文件名全都在该文件中，看注释或者单词名即可。篇幅原因就不粘贴出来了。

/usr/share/beef-xss/extensions/metasploit/config.yaml

在第二张截图是msf扩展部分，其中端口是msf默认的api接口，beef会和这个端口通讯，其中账户和密码也是默认的，可以修改，也可以不改。后续在msf加载的时候同步修改即可。最下面一部分需要根据自己的msf安装路径进行填写，需要让beef找到msf。

修改配置完成之后需要先打开msf，执行load msgrpc ServerHost=XXXXX Pass=XXXXX,加载成功后打开beef，beef默认需要ROOT执行，msf也必须以ROOT执行，我在不同权限启动的时候出现了报错，可能会无法访问msf的api接口，最新版msf以ROOT权限启动使用kali系统自身的postgresql。

Typhoon有感

发表于 2021-01-06 更新于 2021-01-07 分类于 靶机
本文字数： 1.9k 阅读时长 ≈ 2 分钟

​ 记录了 typhoon 靶机 的多种利用姿势，某次发现苹果吃不到苹果的有感而发，因此较多篇幅描述了redis未授权漏洞利用方法。发现漏洞和成功利用漏洞不是一回事，实战中的复杂情形可能自己人都不熟悉自己做了什么策略。

信息收集

​ 查看本机虚拟机网卡，使用arp-scan进行同网段存活探测。当然nmap -sP 进行icmp存活也都可以。

​ nmap进行Top-3000端口的扫描探测。

Redis未授权

SSH免密登陆

​ 生成SSH公私钥对，将公钥放入redis的key:value中，执行save写入**$HOME/.ssh/authorized_keys**,需要有对应的写入权限，需要有对应的目录。如果Redis是ROOT用户起的进程，写入权限是都没有问题的，如果不是需要猜用户名。在实战中会遇到严格基线要求的情况，即便redis以ROOT起来，但是/etc/ssh/sshd_config中禁止了ROOT登陆，写入公钥后也是没办法登陆的。

​ 加入“\n”的目的是为了不和脏数据混在一起

​ 配置文件将permitrootlogin修改为no则无法免密登陆

写入webshell

​ 若服务器中存在web服务，而且知道绝对路径可以通过上述方式进行一句话木马写入，过程雷同。但是以ROOT进程起来的redis写入的文件属主和属组也是ROOT，这就涉及到了服务器账户默认文件的umask设置，在基线要求严格的系统中ROOT创建的文件，非属主属组的用户是没有权限的，默认是“660”.以本文为例，创建一句话木马文件后需要增加文件的读取权限，因为web进程是www-data用户起来的。

​ 将R权限增加后可以连接webshell

计划任务利用获取shell

​ 需要redis以ROOT用户起进程，否则没有权限写入/var/spool/cron/crontabs/，其实常用的crontab -e就是在这个目录下生成一个相同用户名的文件，文件中就是计划任务的内容。我们利用的方式和上述一样，去写同名用户的文件然后cron服务去执行我们写入的命令。

​ 非ROOT用户虽然也可以用crontab -e建立自己的计划任务，但是非ROOT用户不能写这个目录

​ 当存在漏洞的服务器无法写入公私钥匙，没有web服务，而且不出网，如果是ROOT起来的还能顽抗一下，用crontab写计划任务执行一些操作。下面列出了使用nc进行本地转bash的操作，如果nc没有**-e**选项也可以利用管道符进行操作，不过在实战中的效果就不知道了，一些严格的服务器禁止起监听端口，访问敏感二进制文件。在$HOME/.bashrc这个文件其实也可以写一个alias的别名，利用一下下，不过得人家操作才行,这个文件有R权限也可以生效alias……(自己瞎猜瞎搞的勿喷)

redis漏洞利用还有主从复制，这个没有去搞，主从复制时利用扩展植入恶意的.so文件，实现命令执行。这些可以参考网上大佬们的文章。

SSH枚举-爆破密码-SUDO滥用提权

​ 对SSH端口进行协议识别，然后使用msf的username枚举模块发现有效用户，然后hydra爆破有效密码，最后使用SUDO滥用提权到ROOT，SUDO是指普通用户可以使用ROOT权限执行某些命令，以sudo调用su，这样输入sudoers用户的密码即可，sudo bash，都可以使用sudo -l可以查看。

FTP爆破-上传webshell-内核漏洞提权

​ 针对FTP21端口使用hydra进行爆破，登陆之后在web目录写入php木马，登陆之后查看内核版本信息利用漏洞提权编译之后成功执行为ROOT。

phpmoadmin-账号密码泄露-crontab提权

​ 访问web服务下的robots.txt发现敏感目录，访问之后获得typhoon用户的账户密码，然后在当前用户的可写脚本中找到了ROOT用户执行的但是当前用户可写的脚本文件，我们在这个可写脚本中增加恶意命令，当ROOT执行后成功反弹ROOT-SHELL.find / -perm -2 -type f 2> / dev / null查找可以写的文件.

tomcat默认密码-上传war包getshell

​ tomcat默认管理页面使用默认密码tomcat登录之后可以部署恶意war包获取shell，msf有可以直接使用的exp模块。

lotucms-eval危险函数-命令执行

​ 使用nikto获取目标信息，然后访问目标站点，查看页面html源码对已知信息搜索然后利用msf中已有的对应exp模块获取shell。

durpal8可利用漏洞getshell
